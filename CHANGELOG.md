## [4.4.0] ‚Äî 2026-06-25

### Added
- **`--new="idea"` flag** (`setup.sh`) and **`-New "idea"`** (`setup.ps1`) ‚Äî new project bootstrap mode:
  - Injects the project idea into `.readmeAI` `PROJECT IDENTITY ‚Üí Name` field.
  - On first AI session, the AI reads the idea and runs the **New Project Protocol** (see below).
  - Usage: `bash setup.sh --new="task manager with real-time collaboration"` (or pipe from curl)
- **New Project Protocol** in `.readmeAI` FIRST-TIME SETUP ‚Äî new STEP 1B activated when:
  - The directory has no config files (package.json, go.mod, etc.) ‚Äî empty project.
  - In this mode the AI: reads the idea from PROJECT IDENTITY (or asks if blank), picks the
    best-fit stack from the STEP 2 table with justification, gets confirmation, fills all
    sections from the recommendation instead of the codebase, and offers to scaffold the
    initial file structure automatically.

## [4.1.0] ‚Äî 2026-06-25

### Fixed
- **[CRITICAL] `.readmeAI` overwritten on every run** ‚Äî now skips download if file exists, preserving user content. `-Upgrade` still refreshes it.
- **[CRITICAL] `setup.ps1` fails when run as `.\setup.ps1` on PS 5.1** ‚Äî added UTF-8 BOM so PowerShell 5.1 reads the file in the correct encoding instead of falling back to Windows-1252.
- **[CRITICAL] `-Upgrade` always showed "Current: unknown"** ‚Äî variable name collision between `$m` (VT console mode `[uint32]`) and `$m` (Select-String result). Renamed VT variable to `$mMode`.
- **[CRITICAL] `.sh` files written with UTF-8 BOM** ‚Äî `Set-Content -Encoding utf8` in PS 5.1 adds BOM which breaks bash shebang. Now uses `[System.IO.File]::WriteAllText` with `UTF8Encoding($false)` for `readmeai-sync.sh` and the git post-commit hook.
- **`-Upgrade` saved script to disk** (UTF-8 without BOM, same parse failure) ‚Äî now uses `Invoke-Expression (Invoke-WebRequest ...).Content` to execute in-memory, bypassing file encoding issues.
- **`-Health` gave false high scores on blank template** ‚Äî regex for QUICK REFERENCE, DOMAIN RULES, and SYMBOL INDEX was matching all table rows/list items across the entire file. Now scoped to each specific section using regex extraction. Blank template scores correctly low.
- **`-Update` was identical to `-Detect`** ‚Äî now actually removes the old TECH STACK section before re-adding the refreshed one.
- **`-Validate` was missing Continue and Aider** ‚Äî added `.continue\rules\readmeai.md` and `.aider.conf.yml` to the tool checklist.
- **Cosmetic: "Claude Code hooks ->"** ‚Äî changed to use `‚Üí` like all other summary lines.

## [4.0.0] ‚Äî 2026-06-25

### Added
- **`--upgrade` flag** (`setup.sh`) ‚Äî one command to upgrade to the latest ReadMeAI version. Downloads the latest `setup.sh` from GitHub and re-runs it with `--all`, updating `.readmeAI`, all integration files, and hooks.
- **`-Upgrade` flag** (`setup.ps1`) ‚Äî Windows equivalent. Downloads the latest `setup.ps1` to a temp file and re-runs it with `-All`.
- **Version-behind notification** ‚Äî `--sync` and `--health` (both bash and PowerShell) now fetch the remote version (3s timeout, no cost) and print an upgrade hint if a newer version is available: `‚¨Ü  ReadMeAI v4.0 available (you have v3.6) ‚Üí Upgrade: bash setup.sh --upgrade`.
- `check_version()` helper function in `setup.sh` ‚Äî reusable, called at end of `--sync` and `--health` modes.
- `Invoke-VersionCheck` function in `setup.ps1` ‚Äî PowerShell equivalent.
- Usage comments updated in both files to document `--upgrade` / `-Upgrade`.

### Changed
- Version bumped to v4.0 in `setup.sh`, `setup.ps1`, `.readmeAI` header, `.readmeAI` footer, README badge.

## [3.9.0] ‚Äî 2026-06-25

### Added
- **`‚ùå ERROR PATTERNS` section** (new) ‚Äî table of confirmed AI mistakes with symptom ‚Üí root cause ‚Üí correct approach. Lives between KNOWN ISSUES and AI NOTES. Prevents the AI from repeating the same class of mistake across sessions.
- **`Deprecated / Renamed` subsection in DOMAIN RULES** ‚Äî anti-hallucination anchors listing dead APIs/symbols alongside their replacements. Prevents the AI from reaching for removed functions.
- **`Do NOT touch this session` in SESSION STATE** ‚Äî frozen areas (under migration, owned by another team member). AI stops and flags if asked to touch these.
- **`Branch / workspace` line in SESSION STATE** ‚Äî branch name, uncommitted files, open PR. Recovers exact workspace state in one line on hot restart.
- **`Environment variables` table in optional ENVIRONMENT section** ‚Äî exact var names, required/optional, example (non-secret). Prevents AI hallucination of wrong env var names.
- Version bumped to v3.9 in header, template footer, README badge
